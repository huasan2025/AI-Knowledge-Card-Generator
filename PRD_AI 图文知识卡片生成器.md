# PRD｜AI 图文知识卡片生成器（Cardify AI）

## 1. 产品背景（Background）

在学习和内容创作过程中，用户经常会产生高质量的**金句、读书笔记或个人思考**，但在将这些内容转化为**可公开分享的图文内容**时，通常会遇到以下问题：

* 不知道应该拆成几张卡片
* 不清楚每张卡片的侧重点
* 不会排版或不想花时间设计
* 从“想法”到“可发布内容”的转化成本高

随着大模型能力的发展，AI 已经具备**理解内容结构、提炼观点并生成结构化表达**的能力。本项目尝试利用 AI，将“内容理解 + 内容策划 + 基础视觉呈现”整合到一个极简网页工具中，帮助学习者和创作者**节省时间、提高输出效率**。

---

## 2. 产品目标（Goals）

### 核心目标

* 将用户输入的文本内容，快速转化为**多张可用的图文知识卡片**
* 在生成内容前，**由 AI 自动判断内容类型并给出生成策略建议**
* 提供一个**已部署、可访问的 AI + Web 产品原型**

### 非目标（Out of Scope）

* 不做用户登录与账号系统
* 不做内容长期存储
* 不追求复杂视觉设计或图片生成
* 不支持文件上传（如 PDF、图片）

---

## 3. 目标用户（Target Users）

### 主要用户

* 学习者（读书、上课、自学）
* 内容创作者（公众号、社交媒体、知识型创作者）

### 用户特征

* 有内容输入，但缺乏表达与排版动力
* 希望 1 次输入 → 多个可用输出
* 更关注效率，而非复杂编辑能力

---

## 4. 使用场景（Use Cases）

### 场景一：读书笔记转化

用户在读书过程中记录了一段笔记，希望将其转化为几张适合分享的知识卡片。

### 场景二：金句或观点输出

用户有一句成熟观点，希望将其拆解成多张卡片，用于社交媒体发布。

### 场景三：个人思考沉淀

用户写下一段思考，希望 AI 帮助其整理结构，并形成可复用的表达。

---

## 5. 产品核心价值（Value Proposition）

* **从“内容”到“表达”的自动化转化**
* **AI 不仅生成内容，还参与决策**
* **极低使用成本（无需注册、无需学习）**

---

## 6. 功能需求（Functional Requirements）

### 6.1 输入模块

**功能描述**

* 提供一个多行文本输入框
* 用户可输入：

  * 金句
  * 读书笔记
  * 思考感悟

**交互要求**

* 支持粘贴长文本
* 显示字符数（可选）

---

### 6.2 AI 内容识别与策略建议（核心功能）

AI 需要完成以下判断与输出：

#### 1）内容类型识别

自动判断输入文本属于以下哪一类：

* 金句 / 观点型
* 笔记 / 知识型
* 思考 / 方法论型

#### 2）生成策略建议

* 建议生成的卡片数量（1–4 张）
* 每张卡片的功能定位，例如：

  * 核心观点
  * 解释说明
  * 延展思考
  * 行动建议

#### 3）卡片内容生成

为每一张卡片生成：

* 标题（1 行）
* 要点列表（2–4 条）

---

### 6.3 输出模块（卡片展示）

#### 卡片比例（MVP）

* 1:1（正方形）
* 4:5（竖版知识卡片）

#### 卡片样式（MVP）

* 极简白底风格
* 深色沉稳风格

#### 展示要求

* 每张卡片独立展示
* 支持切换比例与样式（同一内容，不重新调用 AI）

---

## 7. AI 输出结构（数据规范）

AI 返回统一的 JSON 结构，便于前端解析：

```json
{
  "contentType": "思考 / 方法论型",
  "suggestedCards": 3,
  "strategy": [
    "核心观点",
    "解释说明",
    "行动建议"
  ],
  "cards": [
    {
      "title": "学习不是信息积累",
      "points": [
        "关键在于形成结构",
        "结构决定可复用性",
        "输出是最好的检验"
      ]
    }
  ]
}
```

---

## 8. 页面结构（Information Architecture）

```
- 页面标题
- 项目简介
- 输入区（文本框 + 生成按钮）
- AI 分析结果区
  - 内容类型
  - 卡片生成建议
- 卡片展示区
  - 卡片列表
  - 样式 / 比例切换
```

---

## 9. 技术方案（Technical Solution）

### 前端

* Next.js（App Router）
* HTML / CSS / JavaScript
* Tailwind CSS（或基础 CSS）

### AI 能力

* OpenAI API 或 Gemini API
* Prompt 驱动内容理解与结构化输出

### 部署

* Vercel
* 环境变量管理 API Key

---

## 10. MVP 交付标准（Acceptance Criteria）

* 页面可正常访问（公网 URL）
* 输入文本后可成功生成卡片
* AI 输出包含：

  * 内容类型判断
  * 卡片数量建议
  * 多张结构化卡片
* 至少支持 2 种比例与 2 种样式
* 无严重交互或渲染错误

---

## 11. 项目风险与应对

| 风险       | 应对方式         |
| -------- | ------------ |
| AI 输出不稳定 | 通过 JSON 格式约束 |
| UI 实现复杂  | 仅使用基础布局      |
| 时间不够     | 严格限制 MVP 范围  |

---

它不仅是一个技术 Demo，更是一次 **“从用户问题出发，到 AI 产品落地”的完整实践**。
